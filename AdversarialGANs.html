Adversarial Examples Generated with GANs¶

For this section of the project I will detail a new way of generating adversarial examples. This will involve a GAN structure to generate high quality images, but still minimize the odds of a pre trained neural net being correct in its classification. First let's detail the problem that we seek to solve:

What are Adversarial Examples¶

One eccentricity about neural nets is the existence of so-called 'avisarial examples'. These are derived from examples which are classified as a given class $i$, with a high certainty in a pre trained neural net, but with a small perturbation that is usually imperceptible to humans makes the example classified to a different class $j$. We can define this in symbols: if our pre trained neural net is a function $\Phi$, and our original image is a tensor $x$ then we know that $\Phi(x) = c_i$ with a high probability ($c_i$ is the ith class that our neural net classifies to). This means that we would like to generate some noise tensor $r$ such that:

$$ \begin{align*} \Phi(x+r) &= c_j\\ s.t. j &\neq i \end{align*} $$

This is, at a high level, correct, but the devil is in the details. $r$ is an arbitrary tensor right now, so it could, in theory, adjust $x$ so greatly that the image becomes unrecognizable, or qualitatively a different image in general. This is counter productive to our goal since we would like the adversarial image $x+r$ to be as close to the original as possible, meaning we need some way to "reign in" our neural net. [1] offers one such way through its use of constraints on $r$. If we define a loss like this:

$$ min_r c||r||_2 $$

Where $c$ is some scalar, and $||r||_2$ is the L2 norm of $r$. Then we should, in theory, have a low magnitude r that results from the optimization that still makes $\Phi$ misclassify the image (of course there is also a term that penalizes $r$ if it doesn't make $\Phi$ misclassify $x$). This was one of the intriguing properties of neural nets detailed in [1], that you contrive an $r$ that is so small that humans cannot see the difference, but $\Phi$ still makes a mistake. The upshot is that we are able to generate adversarial examples for a given neural net which look almost the same as the original image.

Another highlight in this research field is the viability of so called one-pixel attacks [2]. This is adversarial noise that is generated with another constraint: that the noise itself cannot adjust more than one pixel. This works by using a genetic algorithm which I implemented in my seventh biweekly report. This shows that not only is it possible to generate a noise vector that will change a neural net's classification, but this vector can also be quite sparse.

Problems with Previous Approaches¶

The main problem that I see with previous ideas is that each adversarial example must be rigorously trained and optimized. Each adversarial example requires its own optimization process, which seems as if it might be cumbersome. In an attempt to mitigate this problem I would like to propose a new algorithm for finding adversarial examples: a GAN. Let's start by talking about the general structure:

General Structure of a GAN for Adversarial Example Generation¶

We will create a GAN that can generate the noise vector $r$ based on conditional information ($x$). In general we want two things with this model: for $x+r$ to be convincingly a member of the data (and still convincingly a member of whatever class it was originally), and for $x+r$ to make a pretrained model misclassify this image.

In general this will work as follows: $x$ is input to the generator $G$ -> $G$ outputs a tensor of a same shape as $x$ (this is $r$) -> $r$ is penalized based on its size and density (meaning $r$ should be sparse as in [2]) -> $x+r$ is input into a discriminator to ensure that it is still convincingly a member of the data -> $x+r$ is input into a model and $G$ is penalized if it does not fool the original data. Each of these will involve a loss which I will detail to below, but for now here is a general picture of what I will implement:

Let's start by talking about the structure of each of the neural nets in the picture: $G$, $D$, and $\Phi$

The Structure of each Neural Net¶

Let's start by talking about $\Phi$ since it will be the network we are focused on fooling. This is a pre trained neural net that will be used for classification. We need our GAN to generate noise that, when added to the original image, will fool this network. In our implementation we will take advantage of transfer learning to get state of the art accuracy on MNIST (the data we will be using to fool the network). For this we will use pretrained ResNet-18 as the backbone which we will fine tune. I implemented this in my seventh biweekly report, and I found that it got 99% accuracy on a hold out set.

Now we can talk about $G$ which will just be a normal conditional generator (which takes conditional input as a tensor of shape (1, 28, 28) in the case of MNIST, and outputs an image of the same shape). This will follow the structure of the generator that we found in the first half of this project using a U-Net structure with an ASPP attention module (see the first half of this project for more information on this).

$D$ is simply the discriminator we will use in this set up. I've opted not to use a conditional discriminator despite the possibility of it more tightly regulating that the image generated still looks like the original class. This is because I believe that the penalization on the $r$ tensor generated by $G$ for large and dense inputs should be sufficient to stop the generator from completely changing the original image to something else. Now I would like to talk about the loss functions I will employ:

The Losses¶

We've got four losses to talk about, two on $r$ and two on $x+r$, to start let's talk about the two losses on $r$.

The Size Loss¶

This is a loss on the absolute magnitude of $r$. We want $r$ to be as small in magnitude as we can get it, so we will introduce a loss much like that of [1]: $$ loss_{size} = ||r||_2 $$ By minimizing this we hope that the changes we do make will be minor, keeping the original content of the image intact.

Sparsity Loss¶

This loss is designed to make sure that the $r$ only adjusts a few values. We will take a page from TabNet's [3] book (which I reviewed in the second biweekly report) where we will have a penalty for dense masks. TabNet used this to try and force the network to make a sparse feature selection across its attentive masks, but we will use it to try and make $r$ sparse. In TabNet this was defined as:

$$ \lambda = \sum_{i=1}^{N_{steps}}\sum_{b=1}^B\sum_{j=1}^D-\frac{M_{b,j}[i]}{N_{steps}*B}*log(M_{b,j}[i]+\epsilon) $$

Where $M_{b,j}[i]$ was the $(b,j)$th element of the ith mask in TabNet, $N_{steps}$ was the number of masks, $B$ was the batch size, and $\epsilon$ was a term to make sure that the log was numerically stable. Let's consider why this was effective at encouraging sparse masks in TabNet so that we can use it in our project. We see that, since we have $M_{b,j}[i]$ as a term in the numerator of this expression (and since $\epsilon \neq 0$)), if $M_{b,j}[i]=0$ for all $b$, $j$, and $i$, then this sum will be zero. We can also note that if $M_{b,j}[i] > 0$ then $\lambda$ is negative for $M_{b,j}[i] + \epsilon > 1$ (so we will eventually use a tanh function to constrain our output), and positive for $M_{b,j}[i] + \epsilon < 1$. This means that this loss is actually pushing $M_{b,j}[i]$ to be less than $1-\epsilon$ in this case since the zero solution gives a loss of zero, and the more that $M_{b,j}[i] >0$ is pushed toward zero the smaller the loss will be. Now, $M_{b,j}[i]$ is assumed to be positive in the paper and less than one, which makes this loss an easy choice, but for our purposes $r$ needs to be negative for some values. To mitigate this we will need to make some changes to both make this valid for our purposes, and to make sure that it encourages gradient descent to make values zero in both directions. To do this I propose using two different functions: one for a positive value of $r$ (which we are treating as the mask $M[i]$)) and one for a negative value:

For positive values of $r$ ($r[i,j]$) we can apply the function:

$$ loss_{sparse}(r[i,j]) = -r[i,j]*log(r[i,j] + \epsilon), r[i,j] > 0 $$

And for negative values we can use the following slightly adjusted values:

$$ loss_{sparse}(r[i,j]) = r[i,j]*log(abs(r[i,j]) + \epsilon), r[i,j] < 0 $$

Then for all $i = 1...H$ and $j = 1...W$ we can define the complete sparse loss as:

$$ loss_{sparse} = \sum_{i=1}^H\sum_{j=1}^Wloss_{sparse}(r[i,j])/(H*W) $$

Note that the division by $H*W$ is just to take the mean of all of these losses. This should encourage sparse $r$ tensors in both directions (whether $r[i,j] > 0$ or $r[i,j] < 0$). Now we can talk about the discriminator loss and the adversarial loss:

Discriminator Loss¶

Per our observations at the start of this project we will not employ Wasserstein loss, and instead we will use the conventional cross entropy loss for the discriminator:

$$ loss_{fine}(D,G) = log(D(y_i)) + log(1-D(G(x_i,y_i))) $$

We won't use a patch discriminator since we are just trying to get a sense of whether the output image $x+r$ looks real or not (not necessarily whether all aspects of it are perfect).

Adversarial Loss¶

This loss will be a numerical gauge of whether or not $x+r$ will make $\Phi$ misclassify an example. For this loss we will seek to minimize the correct result (rather than maximize the classification of some other classes' probability output in $\Phi$, as some setups like that of [1] will incorporate). I will implement this simply as the negative of the normal cross entropy loss (which will make $G$ attempt to minimize the probability that $\Phi$ is correct, rather than maximize it as would be the usual method of training $\Phi$)

Putting All Of This Together¶

This will give a final loss that is the sum of all of the losses I just described. I will also add a scaling to each term to weigh the relative importance of each loss, though these will have to be found by hand since I don't think autoML is reasonable given my system resources.

        self.convLayers = torch.nn.Sequential(*(list(models.resnet18(pretrained=True).children())[:-3]))

        self.fc2 = nn.Linear(50,50)

        x = self.relu(self.fc1(x))
        x = self.relu(self.fc2(x))
        x = torch.softmax(self.fc3(x), dim = 1)
        return x

"""
AttentiveFuse -- implements an attentive fuse operation as detailed by https://arxiv.org/pdf/1901.06322.pdf

@params - c_in: the number of input channels (it will output 1/2 this number, since that will keep the number of filters constant
                in the large network)
          c_out: The number of output channels in the convolutional layer
"""
class AttentiveFuse(nn.Module):
    def __init__(self, c_in, c_out):
        super(AttentiveFuse, self).__init__()
        self.c_in = c_in
        self.c_out = c_out
        #----------
        #The first convolutional layers:
        #----------
        self.convLayers1 = nn.Sequential(nn.Conv2d(self.c_in, self.c_out, 3, padding = 1),
                                        nn.ReLU(),
                                        nn.BatchNorm2d(self.c_out),
                                        nn.Conv2d(self.c_out, 2*self.c_out, 3, padding = 1),
                                        nn.ReLU(),
                                        nn.BatchNorm2d(2*self.c_out),
                                        nn.Conv2d(2*self.c_out, self.c_in, 3, padding = 1),
                                        nn.Sigmoid(),
                                        nn.BatchNorm2d(self.c_in))
        #----------
        #The second convolutional layers:
        #----------
        self.convLayers2 = nn.Sequential(nn.Conv2d(int(self.c_in/2), self.c_out, 3, padding = 1),
                                        nn.ReLU(),
                                        nn.BatchNorm2d(self.c_out),
                                        nn.Conv2d(self.c_out, int(self.c_in/2), 3, padding = 1),
                                        nn.ReLU(),
                                        nn.BatchNorm2d(int(self.c_in/2)))
    def forward(self, x, y):
        #----------
        #concatinate the two inputs
        #----------
        cat = torch.cat((x,y), dim = 1)
        
        #----------
        #apply the first convolutional layers
        #----------
        cat = self.convLayers1(cat)
        
        #----------
        #separate the attentive filters and multiply pointwise
        #----------
        x_att = cat[:,0:int(self.c_in/2),:,:]
        y_att = cat[:,int(self.c_in/2):cat.shape[1],:,:]
        
        x = x*x_att
        y = y*y_att
        
        #----------
        #add both of the inputs (scaled by attention) together
        #----------
        out = x+y
        
        #----------
        #Run out through the final convolutional layers
        #----------
        out = self.convLayers2(out)
        return out

"""
SpatialPooling -- Implements attentive spatial pooling as defined by https://arxiv.org/pdf/1901.06322.pdf

@params - c_in - input channels
          c_out - number of channels in the inbetween networks
          dilationRates - what dilation rates should we use in our convolutions for ASPP
"""
from torch.nn.parameter import Parameter
class SpatialPooling(nn.Module):
    def __init__(self, c_in, c_out = 32, dilationRates = [3,6,12,18]):
        super(SpatialPooling, self).__init__()
        #----------
        #Initialize parameters
        #----------
        self.dilationRates = dilationRates
        self.c_in = c_in
        self.c_out = c_out
        self.atrousConvs = nn.ModuleList([]) #Keeps track of the set of atrous convolutions
        self.fuses = nn.ModuleList([]) #Keeps track of the fusing modules
        
        #----------
        #Populate atrous Convs and fuses modules lists
        #We need 1 less fusion module than we have atrous convolution representation (so we have an if statement to take care of this)
        #Each atrous conv in this module will follow the form atrous conv -> relu -> batch normalization
        #----------
        for i in self.dilationRates:
            if i != self.dilationRates[0]:
                self.fuses.append(AttentiveFuse(self.c_out*2, 32))
            self.atrousConvs.append(nn.Sequential(nn.Conv2d(self.c_in, self.c_out, 3, padding = i, dilation = i),
                                             nn.ReLU(),
                                             nn.BatchNorm2d(self.c_out)))
        #----------
        #In order to apply the residual connection we will need to reduce the dimension of the output to have the same number of filters as
        #the input
        #----------
        self.convReduce = nn.Sequential(nn.Conv2d(self.c_out, self.c_in, 1),
                                        nn.ReLU(),
                                        nn.BatchNorm2d(self.c_in))
        
        #----------
        #Initialize the gamma in the residual scaling
        #----------
        self.gamma = Parameter(torch.zeros(1,1))
        self.gamma = nn.init.xavier_uniform_(self.gamma)
    def forward(self, x):
        #----------
        #Save the original x for the skip connection
        #----------
        x_mem = x.clone()
        
        #----------
        #Do the atrous convolutions, keeping track of each one in a list
        #----------
        savedConvs = []
        for i in range(len(self.atrousConvs)):
            savedConvs.append(self.atrousConvs[i](x))
            
        #----------
        #Fuse together all of the atrous convolutions
        #----------
        for i in range(len(self.fuses)):
            if i == 0:
                out = self.fuses[i](savedConvs[i], savedConvs[i+1])
            else:
                out = self.fuses[i](out, savedConvs[i+1])
        
        #----------
        #Reduce the number of filters in out so that it is the same as that of the input
        #----------
        out = self.convReduce(out)
        
        #----------
        #Create a residual connection with a learned scaling
        #----------
        out = out*self.gamma + (1-self.gamma)*x_mem
        return out
"""
encoderBlock - defines a single encoder block for the generator

@params: Cin - int, the number of input channels
         Cout - int, the number of output channels
         batchNorm - bool, tells us if we should use batch normalization for this layer or not
@methods: forward: defines the forward pass of the model
"""
class encoderBlock(nn.Module):
    def __init__(self, Cin, Cout, batchNorm = True):
        super(encoderBlock, self).__init__()
        self.conv = nn.Conv2d(Cin, Cout, 4, stride = 2, padding = 1) #reduces dim by 1/2
        self.BN = nn.BatchNorm2d(Cout)
        self.leReLU = nn.LeakyReLU(.2)
        self.batchNorm = batchNorm
    def forward(self, x):
        if self.batchNorm:
            x = self.leReLU(self.BN(self.conv(x)))
        else:
            x = self.leReLU(self.conv(x))
        return x
    
"""
decoderBlock - defines a decoder block for the generator

@params: Cin - int, the number of input channels
         Cout - int, the number of output channels
         useDrop - bool, whether or not to use a dropout layer
         batchNorm - bool, whether or not to use batch normalization
@methods: forward(self, x): defines the forward pass of the model
"""
class decoderBlock(nn.Module):
    def __init__(self, Cin, Cout, useDrop = True, batchNorm = True):
        super(decoderBlock, self).__init__()
        self.tConv = nn.ConvTranspose2d(Cin, Cout, 4, stride = 2, padding = 1) #upsamples by 2
        self.BN = nn.BatchNorm2d(Cout)
        self.drop = nn.Dropout(.5)
        self.useDrop = useDrop
        self.batchNorm = batchNorm
    def forward(self, x):
        if self.useDrop and self.batchNorm:
            x = self.drop(self.BN(self.tConv(x)))
        elif self.batchNorm:
            x = self.BN(self.tConv(x))
        else:
            x = self.tConv(x)
        return x

"""
Generator - defines a generator for our image translations task, expects images to be 512x512

@params - None
@methods: forward(self, x): Defines the forward pass of the model
"""
class Generator(nn.Module):
    def __init__(self):
        super(Generator, self).__init__()
        #-----------
        #U Net part
        #-----------
        #define our encoder blocks:
        #28x28
        self.enc1 = encoderBlock(1, 128, False)
        #14x14
        self.enc2 = encoderBlock(128, 128)
        #7x7
        #this is probably small enough to start adding decoder blocks (I was not able to make the full model due to RAM issues)
        self.dec1 = decoderBlock(128,128, False)
        #14x14
        self.dec2 = decoderBlock(128*2, 1, False, False)
        #28x28
        
        #-----------
        #ASPP Part
        #-----------
        self.ASPP = SpatialPooling(1)
        self.finalConv = nn.Sequential(nn.Conv2d(2,1,1),
                                       nn.BatchNorm2d(1))
        self.relu = nn.ReLU()
        self.tanh = nn.Tanh()
    def forward(self, x):
        x_mem = x.clone()
        #----------
        #ASPP on the input
        #----------
        xASPP = x.clone()
        xASPP = self.ASPP(xASPP)
        
        #list of the inputs to use in U-net
        temp = []
        
        #----------
        #contracting path
        #----------
        x = self.enc1(x)
        #append for use in U-net
        temp.append(x)
        x = self.enc2(x)
        
        #----------
        #dilating path
        #----------
        x = self.dec1(x)
        x = F.relu(torch.cat((x, temp[0]), dim = 1))
        
        #output
        x = self.relu(self.dec2(x))
        x = self.tanh(self.finalConv(torch.cat((x,xASPP), dim = 1)))
        x_mem = x+x_mem
        return x_mem, x

    def forward(self, x):

        x = self.C128(x)
        x = x.view(-1,64*7*7)
        x = torch.sigmoid(self.fc1(x))
        return x

Now we can implement the sparsity loss (since this one will need to be a custom function):

In [47]:
"""
NegativeSLoss -- Calculates the sparsity loss for a tensor of negative values

@params: - x, a tensor of only negative values
"""
def NegativeSLoss(x):
    eps = 10**-6
    loss = x*torch.log(torch.abs(x) + eps)
    return loss
"""
PostiveSLoss -- Calculates the sparsity loss for a tensor of positive values

@params: - x, a tensor of only positive values
"""
def PostiveSLoss(x):
    eps = 10**-6
    loss = -x*torch.log(x + eps)
    return loss

"""
SparseLoss -- Calculates the sparse loss for a tensor, returns a single value

@params: - x, a tensor of values that we would like the sparse loss of
"""
def SparseLoss(x):
    nLoss = NegativeSLoss(x*(x<0))
    pLoss = PostiveSLoss(x*(x>0))
    loss = torch.mean(nLoss+pLoss)
    return loss

            #Find the classification loss

We will analyze thes ablation studies in more depth in the summary document for this section of the project, but for now it appears that the sparsity loss is important for generalizability, the size loss is important for making sure the images are still distinguishable from noise, and it would appear that the discriminator may not play an extremely important role in this setup.

Some Problems with this Approach¶

I would like to quickly detail some problems that I had with this approach. The first is that there is nothing stopping the images generated by the generator from being invalid images. This means that the image would contain values less than negative 1 or values greater than 1. I'm hoping that the sparsity loss and size loss will make this less of a problem, and the invalid images should be easy for the discriminator to classify as fake, but this is not a rigorous way of mitigating this concern, so it is certainly still possible for the model to converge to a solution that only gives invalid images. I also would like to note that the adversarial images generated by this typically affect the digit itself substantially. This is fine in my eyes since I can still easily tell what the digit is, but it is a far cry from the almost imperceptible noise generated from other methods.

Summary/Final Thoughts About Using GANs to Generate Adversarial Images¶

This worked better than I had originally hoped, and it was able to fool $\Phi$ even on a test set of images, which says to me that the generator did learn where weaknesses were in $\Phi$'s structure. This is advantageous over other methods because it does not need to optimize for each image that it would like to generate an adversarial example for, and instead can be trained once and still fool a pre trained neural net for images that it has never seen before.
